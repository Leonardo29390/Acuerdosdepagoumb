## Documentación del Proyecto

### Propósito y ventajas de la estructura utilizada

La estructura utilizada en el ejercicio grupal se basa en el uso de **HTML5 semántico**, organizando el contenido mediante etiquetas como `header`, `nav`, `main`, `section`, `article`, `aside` y `footer`.  
El propósito principal de esta estructura es lograr un sitio web **ordenado, claro y fácil de comprender**, tanto para los desarrolladores como para los usuarios finales.

Entre las principales ventajas de esta estructura se encuentran:
- Facilita el **trabajo colaborativo**, ya que cada sección del sitio tiene una función clara.
- Mejora el **mantenimiento del código**, permitiendo realizar cambios sin afectar otras partes del sitio.
- Incrementa la **accesibilidad web**, favoreciendo a usuarios con tecnologías de asistencia.
- Optimiza el **posicionamiento en buscadores (SEO)** gracias al uso correcto de etiquetas semánticas.

### Importancia de la separación entre cliente y servidor en aplicaciones web

La separación entre cliente y servidor es fundamental en las aplicaciones web modernas, ya que permite dividir responsabilidades de forma eficiente.  
El **cliente**, representado por el navegador, se encarga de mostrar la información e interactuar con el usuario mediante tecnologías como HTML, CSS y JavaScript.  
El **servidor**, por su parte, gestiona la lógica del negocio, el almacenamiento de datos y el procesamiento de las solicitudes.

Esta separación aporta múltiples beneficios:
- Mejora la **escalabilidad**, permitiendo que el sistema crezca sin afectar su funcionamiento.
- Facilita el **mantenimiento y la actualización** de cada parte de manera independiente.
- Aumenta la **seguridad**, al centralizar la gestión de datos sensibles en el servidor.
- Permite reutilizar el mismo servidor para distintos clientes o interfaces.

En el ejercicio desarrollado, esta separación se refleja en el uso del frontend como base del sistema, respetando las buenas prácticas de la ingeniería web.
